Posted on 2015-07-25   |   In 前端   |  

前言

总是写前言是因为照顾像我这样的懒人，万一文章长了就不想看了，所以相当于是写一个概述放在前头，让人决定要不要看下去的。

之前报名了百度 IFE 暑假训练营，形式是组队完成任务，然而自己认识的朋友不多，初涉前端圈子，自然没有找到认识的人组队刷副本。然后就加到群里自由组队，一个队伍收留了我，后来证实这个团队是不靠谱的，这里不多展开。然后我也不可能一个人去搞团队项目（实际上还是有人这么干的…)，所以就回到了“每逢期末考试的不明烦躁期”开的一个坑——之前还没听说 IFE 训练营的时候也是打算暑假要苦练前端技术，首先还是要练习好基本功“重构”，所以就网上找一些网页设计原型图开始切。然后这个项目完成度不是很高，因为我之前还打算靠着意淫把其他页面按同样风格做出来来着…但我还是学到了一些（遇到了一些坑）的。故写一篇文章来记录一下这次练习，下面涉及一些极其简单的前端知识，没兴趣的还是不要往下看了。

字体方面由于深知字体加载的痛苦，个人博客就是使用了自定义字体所以加载比较慢，所以就没有使用原型图里的字体，多采用微软雅黑和 Arial 字体。

除了头图和 logo 图是 PNG 格式图片，其他均采用 SVG 矢量格式，这是因为源图作者标明是适配 Retina 屏幕的 （当然我除了手机并没有其他高 DPI 设备可以做测试，但效果应该不至于好差）。

坑1，叫你装逼不用 jQuery

在下面的数据 Analytics 区域，有一个类似的下拉菜单，不一样的是它的内容应该会有12个，一月到十二月，那么总不能把下拉列表弄得好长好长吧，这时候就应该把 overflow set 为 auto 但是这时候又会出现了不协调的滚动条。

一开始想着把滚动条设置为不显示就可以了，但是支持修改原生滚动条的浏览器只有 IE 和 Chrome, 这时我就想着自己写一个滚动的效果应该不难吧。主要就是把外层元素样式设置为 overflow:hidden;, 内里列表设置为 position:absolute;, 然后绑定 mouseover, mouseover, touchstart, touchend 这几个关键的事件和通过 scroll 事件来控制内里列表的 top 样式属性值，这听起来真的不难，然后我开始尝试着造轮子…

最后，我引入了 jQuery 库和一个基于它的现成的自定义滚动栏插件，一开始我只是打算写纯碎的 JS 但是我还是屈服了，请不要问我原因是什么。

最后实现的效果还是挺满意的，就是仅仅这么一个要求就引入库有点不值得，果然还是我太菜了。

SVG 交互、动画

网页里面使用了三个主要的 SVG 用作数据展示，将各部分从 PS 文档转为 AI 并不困难，但是我要重新调整比例什么的花费了一些时间，还有它原来的图表也画的并不精确需要微调。

一开始我就几乎想好了动效是怎样和交互应该是怎样，然后在完成基本的布局就开始写。

在淡入淡出动画方面由于 jQuery 自带的不是非常流畅，因此引入 Velocity.js 库来优化（尽管还是会卡顿…)。

时间轴的动效很简单，就是由无到有嘛，使用 stroke-dashoffset 和 stroke-dasharray 两个属性就可以控制 SVG 路径的描边效果， 使用纯 CSS3 也可以完成。后来我引入了 Snap.svg 一个由 Adobe 开发的 SVG 动画库，然后重新计算了时间使用 JS 做出了动效，因为这里不需要太精确，所以时间计算非常粗略。在自适应方面设置最大宽度为 1440px 避免和页面比例失衡，然后当小于 1024px 时已经很难看清楚了，

坑2，不应该那么复杂才对

首先在原型图里的柱形图是用描边来画的，然后我神经质地把它转化为矩形，后来发现坑了自己。而且 AI 自身自带图表功能我之前也并不清楚，就使用了非常愚蠢的方式绘制图形，以至于使用了两层填充来绘制一条图形。其实这也只是自己挖的一个小坑。

真正的坑在于原型图里柱状图只有13条，而10月有31天，多出来的可不能不管，于是我想到了拖动的方式来交互，然后就使用了 Snap.svg 来开发交互。过程挺不顺利，首先它偏偏要在两边加上淡入效果，也就是边上的透明度比较低，那动起来也要计算在内了。后来我写着写着，觉得为什么不把两边直接用一层渐变覆盖起来呢，然后试了一下，发现效果不堪入目：

可以看到明显的色阶条纹，什么，你看不出来？

当然完全可以就弄两个颜色遮罩，但我已几近乎完成这个交互了，再考虑到颜色变化的自然过渡，最终还是没有去折腾（主要还是懒）。

然后当我弄得差不多的时候发现这样普通的拖动似乎太平常，而且不太流畅，我就想到了要加入惯性滑动…结果搞了几个小时都没有弄出来像样的效果还是放弃了。果然还是眼高手低，仅仅是像平时手机浏览的那种滑动惯性都做不出来。貌似把它改成滚动的就应该可以在手机上面有滚动惯性了，我也没有再深入研究，反正现在的效果是不太令人满意。

教训：主要还是在计算边界情况上面和自适应上面要注意处理的情况比较多，需要全面分析临界状况，否则容易出现错误。
